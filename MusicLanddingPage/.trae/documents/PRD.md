# BawMusic Router 产品需求文档 (PRD)

## 1. 项目概述

**BawMusic Router**（目录名 `MusicLanddingPage`，用户保留的拼写）是一个零交互智能入口落地页。
- 两个 BawMusic 部署点：
  - `https://bawmusic.top`（主源 / 常规入口）
  - `https://eo.bawmusic.top`（边缘优化入口 / 阿里云 ESA）
- 用户访问该页时，自动 ping 两个目标，从可用节点中挑延迟最低的，秒级跳转。
- 完全前端，无后端，无 API Key。部署到任何静态托管（GitHub Pages / Vercel / 阿里云 ESA / CloudFlare Pages）即可。

## 2. 用户场景

| 场景 | 描述 |
| ---- | ---- |
| 入口流量 | 用户通过 `www.bawmusic.top` / `router.bawmusic.top` / 分享链接 / 收藏夹进入 BawMusic |
| 自动选路 | 页面加载即测速 1~3 次，2 秒内完成决策 |
| 自动跳转 | 默认倒计时 2.5 秒后 `window.location` 跳到更快节点 |
| 手动兜底 | 用户可点链接手动前往任一节点（防测速抖动 / 自动跳转被浏览器拦截） |
| 测速失败 | 两个都不可达时，显示错误 + 手动链接，不自动跳 |
| 移动端 / 弱网 | 同逻辑；弱网会显著放大失败概率，跳转按钮需醒目 |

## 3. 核心功能

1. **并行 ping**：用 `fetch(url, { mode: 'no-cors', cache: 'no-store' })` + `performance.now()` 计时，2 秒超时。
2. **多次取最优**：每个目标 ping 2 次（间隔 200ms），取最小延迟；过滤 timeout 节点。
3. **跨域兼容**：`no-cors` 保证从任何静态托管域发起都不会 CORS 失败。
4. **路径透传**：跳转到更快节点时，保留原 URL 的 `pathname + search + hash`（即 `?song=xxx` 分享参数不丢）。
5. **自动跳转 + 倒计时**：2.5 秒倒计时显示在主卡片里，可点取消。倒计时结束前可重新测速。
6. **手动链接**：始终显示两个目标的手动链接，不论结果如何都可见。
7. **预解析**：跳转目标在测速完成后插入 `<link rel="dns-prefetch">` / `<link rel="preconnect">` 提前建连。

## 4. 页面设计

桌面优先，移动端自适应。

```
┌─────────────────────────────────────────────┐
│             BawMusic Router                 │  ← 顶部品牌行
│        正在为你选择最优入口                  │  ← 副标题
├─────────────────────────────────────────────┤
│  ┌────────────────────────────────────┐     │
│  │  bawmusic.top          测速中...   │     │  ← 卡片 1
│  │  ━━━━━━━━━━○○○○  128 ms           │     │
│  └────────────────────────────────────┘     │
│  ┌────────────────────────────────────┐     │
│  │  eo.bawmusic.top      测速中...   │     │  ← 卡片 2
│  │  ━━━○○○○○○○○○○   64 ms ✓         │     │
│  └────────────────────────────────────┘     │
├─────────────────────────────────────────────┤
│        将于 2.5 秒后跳转到 eo.bawmusic.top  │  ← 倒计时 + 取消
│  [ 取消跳转 ]  [ 重新测速 ]                  │
├─────────────────────────────────────────────┤
│  手动选择：                                  │
│  → 前往 bawmusic.top                        │
│  → 前往 eo.bawmusic.top                     │
└─────────────────────────────────────────────┘
```

视觉语言与 BawMusic / BawTV 一致：纯黑 + 玻璃质感 + 系统字体 + 圆角胶囊。

## 5. 非功能需求

- **首屏体积**：HTML+JS+CSS gzip 后 < 30 KB。
- **首屏可见时间**：< 1s（测速和渲染并行）。
- **零依赖**：不引入 React（用原生 DOM + Lit / 直接 template），仅保留样式可选框架；为简化直接用 Next.js 16 静态导出（与 BawMusic 同栈）。
- **可降级**：JS 关闭时显示静态链接列表。
- **无 cookie / 无 localStorage / 无埋点**。

## 6. 验收标准

- [ ] 并行测速两个目标，2 秒内出结果
- [ ] 测速失败 / 超时显示具体错误
- [ ] 倒计时可在 0.5 ~ 5 秒之间可配（默认 2.5）
- [ ] 倒计时期间可取消 / 重新测速
- [ ] 跳转到目标时保留原 URL 的 path / query / hash
- [ ] 桌面 (1280px) 和移动 (375px) 布局无错位
- [ ] Lighthouse Performance ≥ 95

## 7. 范围之外

- 不做用户偏好记忆
- 不做 A/B 分流
- 不做服务端 IP 地理判断（前端拿不到，意义不大）
- 不做埋点 / 分析
